Obtaining Your License

Gurobi Optimizer 5.6 uses the same licenses as version 5.5, so if you already have a Gurobi 5 license, there's no need to obtain a new one. You may need to copy your existing license file to a new location, though. Consult the Quick Start Guide for information on where to store your license file.

If you don't already have a Gurobi 5 license, you will need to visit the Licenses page on the Gurobi website. If you are a commercial user under maintenance, you should see your Gurobi 5 license under Current. If you would like to request a free trial or free academic license, you should request a new license from the Free Trial or Free Academic menu items. Once you have a license on the Gurobi web site, you will need to follow the instructions for obtaining and installing a license in the Quick Start Guide.

Release Notes for Version 5.6

In addition to performance enhancements, Gurobi 5.6 adds new features, Python 3.2 support, R 3.0 support, and increases the size of problem you can solve with our free trial version:

  • Distributed tuning and distributed concurrent MIP: Gurobi Compute Server users can now harness the power of multiple independent machines to do parallel parameter tuning or concurrent MIP solving. You simply provide a list of compute servers, and Gurobi automatically distributes the work among the available machines. Several new parameters have been added to support this new capability:
    • ConcurrentMIPJobs: Indicates how many concurrent MIP jobs to start. You would typically run one job on each available compute server, but you also have the option of running multiple jobs on a single compute server.
    • TuneJobs: Indicates how many tuning jobs to start. Again, you would typically run one job on each available compute server, but you can also run multiple jobs on a single compute server.
    • ServerPool: The list of Gurobi compute servers to use for distributed tuning or distributed concurrent MIP solving.
    • ServerPassword: The access password for the compute servers (if any).
    Please refer to the distributed parallel algorithm discussion in the Gurobi Reference Manual for additional details on these new features.
  • MPS and LP file reading and writing from MATLAB: You can now use the Gurobi MATLAB interface to read an MPS or LP file into a MATLAB matrix, or write a MATLAB matrix into an MPS or LP file.
  • Support for Python 3.2 on Linux and Windows: You can now use Gurobi from Python 3.2. Use the setup.py script included with this distribution to install the Gurobi module into your Python 3.2 installation.
  • Support for R 3.0: You can now use Gurobi from R 3.0.
  • Solve larger problems with our free trial version: The Gurobi free trial now allows you to build and solve models with up to 2000 constraints and 2000 variables.
  • Additional new parameters:
    • Disconnected: MIP models are sometimes made up of multiple, independent sub-models. This parameter controls the strategy used to exploit this structure.
    • PreSOS1BigM and PreSOS2BigM: These new parameters provide user control of presolve SOS linearization.
    • ConcurrentSettings: This command-line only parameter allows you to create concurrent environments (for concurrent MIP) from a set of .prm files.
    Details can be found in the parameters section of the Gurobi Reference Manual.
  • New sync routines for non-blocking optimization: Users of the non-blocking optimization feature in Compute Server should use the new sync routines, available in all supported languages, to sychronize with an optimization that is in progress.

Code Conversion from Version 5.5

Programs that were originally written for version 5.5 of the Gurobi Optimizer require no code changes to use version 5.6. If you'd like to use the new features, some code changes are required.
